## Purpose
This agent assists developers in generating a new page or component in the Next.js project while following a clean architecture:
1. **Logic Separation** – Extracts business logic into custom React hooks.
2. **Server Actions** – Generates server‑side actions (Next.js 13 server actions) for data fetching or mutations.
3. **Type Definitions** – Creates TypeScript types for props, state, and API responses.
4. **Generic Components** – Adds reusable UI components (e.g., form controls) when required.

## Project Specifics
- **Next.js version**: 16.2.1
- **UI library**: shadcn (v4.1.0)
- **Linter**: Biome (v2.2.0)
- **Runtime**: Bun (used for dev scripts and server execution)

## Usage
1. Invoke the agent with a component name, e.g. `create-page-component DashboardPage`.
2. The agent will:
   - Create a new page under `src/app/<name>/page.tsx`.
   - Generate a corresponding hook under `src/hooks/<name>.hook.ts`.
   - Generate a server action under `src/actions/<name>.ts`.
   - Create a types file under `src/types/<name>.ts`.
   - If the page requires form controls, add generic components under `src/components/<name>/`.
3. The agent will prompt for any additional details (e.g., props, API endpoints) if needed.

## Tool Preferences
- **File System** – `create_file`, `apply_patch`.
- **Terminal** – `run_in_terminal` for running `bun run dev` after generation if desired.
- Avoid using `run_task` unless the user explicitly requests to start the dev server.

## Example Prompt
`create-page-component LoginForm`

## Notes
The agent respects the existing project structure and naming conventions. It will not overwrite existing files unless the user confirms.
