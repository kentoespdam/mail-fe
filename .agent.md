name: create-page-component
description: Scaffolds Next.js pages/components using clean architecture (Hooks, Actions, Types, and UI separation).
argument-hint: Component/Page name (e.g., "Dashboard")
---

## Stack & Standards
- **Tech:** Next.js 16.2.1, shadcn (v4.1.0), Bun Runtime.
- **Linting:** Biome (v2.2.0).
- **Architecture:** Strict separation of logic (hooks), data (actions), and UI.

## File Structure Generation
For a given `<name>`, generate:
- **Page:** `src/app/<name>/page.tsx`
- **Logic:** `src/hooks/<name>.hook.ts` (Business logic)
- **Actions:** `src/actions/<name>.ts` (Server actions)
- **Types:** `src/types/<name>.ts` (Interfaces/Types)
- **UI:** `src/components/<name>/` (Generic components/Form controls)

## Tooling & Constraints
- **Preferred Tools:** `create_file`, `apply_patch`.
- **Terminal:** Only use `run_in_terminal` for `bun run dev` if explicitly requested.
- **Safety:** Do NOT overwrite existing files without confirmation.
- **Validation:** Ensure all generated code passes Biome linting and uses TypeScript.

## Example
`create-page-component LoginForm`
